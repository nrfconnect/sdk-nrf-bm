To flash the S115 SoftDevice binary to your device, open the terminal and run the following commands:

1. First, move into the :file:`nrf-bm` working directory:

   .. tabs::

      .. group-tab:: Windows

         .. code-block:: console

            cd C:/ncs/bm_v0.7.0/nrf-bm

      .. group-tab:: Linux

         .. code-block:: console

            cd ~/ncs/bm_v0.7.0/nrf-bm

      .. group-tab:: macOS

         .. code-block:: console

            cd /opt/nordic/ncs/bm_v0.7.0/nrf-bm

#. Then, make sure that your device is erased:

   .. code-block:: console

      nrfutil device erase

#. Finally, flash the SoftDevice binary for the target SoC:

   .. tabs::

      .. group-tab:: nRF54L15

         .. code-block:: console

            nrfutil device program --firmware components/softdevice/s115/s115_9.0.0-3.prototype_nrf54l15_softdevice.hex

      .. group-tab:: nRF54L10

         .. code-block:: console

            nrfutil device program --firmware components/softdevice/s115/s115_9.0.0-3.prototype_nrf54l10_softdevice.hex

      .. group-tab:: nRF54L05

         .. code-block:: console

            nrfutil device program --firmware components/softdevice/s115/s115_9.0.0-3.prototype_nrf54l05_softdevice.hex

   .. note::
      Keep in mind that the flashed SoftDevice variant must be in sync with the board target that you are planning to use in the build configuration of the sample.
      For example, for the ``bm_nrf54l15dk/nrf54l15/cpuapp/s115_softdevice`` board target, you must use the SoftDevice binary :file:`s115_9.0.0-3.prototype_nrf54l15_softdevice.hex` .
