.. note::

   When building for an MCUboot board target, this step is not required.
   With the MCUboot board target, the SoftDevice is signed and included as part of the build process.

To flash the S115 SoftDevice binary to your device:

1. Open the |nRFVSC| terminal

   * In the extension's :guilabel:`Welcome View`, click :guilabel:`Open terminal`. This will open a terminal where you can use the Toolchain commands.

#. Move into the :file:`nrf-bm` working directory:

   .. tabs::

      .. group-tab:: Windows

         .. code-block:: console

            cd C:/ncs/nrf-bm/v0.8.0/nrf-bm

      .. group-tab:: Linux

         .. code-block:: console

            cd ~/ncs/nrf-bm/v0.8.0/nrf-bm

      .. group-tab:: macOS

         .. code-block:: console

            cd /opt/nordic/ncs/nrf-bm/v0.8.0/nrf-bm

#. Then, make sure that your device is erased:

   .. code-block:: console

      nrfutil device erase

#. Finally, flash the SoftDevice binary for the target SoC:

   .. tabs::

      .. group-tab:: nRF54L15

         .. code-block:: console

            nrfutil device program --firmware components/softdevice/s115/s115_9.0.0-3.prototype_nrf54l15_softdevice.hex --options chip_erase_mode=ERASE_NONE

      .. group-tab:: nRF54L10

         .. code-block:: console

            nrfutil device program --firmware components/softdevice/s115/s115_9.0.0-3.prototype_nrf54l10_softdevice.hex --options chip_erase_mode=ERASE_NONE

      .. group-tab:: nRF54L05

         .. code-block:: console

            nrfutil device program --firmware components/softdevice/s115/s115_9.0.0-3.prototype_nrf54l05_softdevice.hex --options chip_erase_mode=ERASE_NONE

   .. note::
      Keep in mind that the flashed SoftDevice variant must be in sync with the board target that you are planning to use in the build configuration of the sample.
      For example, for the ``bm_nrf54l15dk/nrf54l15/cpuapp/s115_softdevice`` board target, you must use the SoftDevice binary :file:`s115_9.0.0-3.prototype_nrf54l15_softdevice.hex` .
